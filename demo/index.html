IDL and C++ parsers combined

Multiple input files, crosslinked. The IDL and C++ code was parsed individually, then linked together. During the link step, names were mapped according to the CORBA C++ language mapping, such that the internal skeleton types are hidden. Files processed with this driver script:

  • Storage, generated from Storage.idl
  • StorableImpl, FactoryImpl, generated from StorableImpl.hh and FactoryImpl.hh
